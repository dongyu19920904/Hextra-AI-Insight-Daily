# 快速启动指南

## ⚠️ 当前状态

- ✅ Hugo Extended 已安装
- ✅ Go 已安装
- ⚠️ 需要下载 Hextra 主题（首次运行）

---

## 🚀 启动步骤（必须按顺序）

### 步骤 1：打开新的 PowerShell 窗口

**重要：** 关闭所有旧的 PowerShell 窗口，打开一个全新的窗口（让 Hugo 和 Go 的 PATH 生效）

### 步骤 2：进入项目目录

```powershell
cd D:\GitHub\Hextra-AI-Insight-Daily
```

### 步骤 3：下载 Hextra 主题（首次运行必须）

```powershell
hugo mod get -u
```

**如果提示网络错误：**
```powershell
# 使用国内镜像
$env:GOPROXY="https://goproxy.cn,direct"
hugo mod get -u
```

**如果仍然失败（GitHub 连接问题）：**
```powershell
# 清理缓存重试
hugo mod clean
hugo mod get -u
```

### 步骤 4：启动服务器

```powershell
hugo server
```

### 步骤 5：访问网站

在浏览器打开：
- **主页**: http://localhost:1313/
- **中文日报**: http://localhost:1313/cn/
- **英文日报**: http://localhost:1313/en/

---

## 🔧 如果网络问题无法解决

### 方案 A：配置 Git 代理（如果有代理）

```powershell
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
hugo mod get -u
```

### 方案 B：手动下载主题（备用方案）

如果 `hugo mod get -u` 始终失败，可以手动下载：

```powershell
# 1. 删除模块配置
# 暂时注释掉 hugo.yaml 中的 module 部分

# 2. 手动克隆主题到 themes 目录
git clone https://github.com/imfing/hextra.git themes/hextra

# 3. 修改 hugo.yaml，改用本地主题
# theme: hextra
```

## 🔍 故障排查

### 问题 1：hugo: command not found

**解决：** 关闭并重新打开 PowerShell 窗口

### 问题 2：failed to download modules

**原因：** 网络连接 GitHub 失败

**解决方案：**
1. 配置 VPN/代理
2. 使用 Go 国内镜像：`$env:GOPROXY="https://goproxy.cn,direct"`
3. 或使用手动下载主题的方案 B

### 问题 3：端口 1313 被占用

**解决：** 使用其他端口
```powershell
hugo server --port 8080
```

### 问题 4：页面显示空白

**检查：**
```powershell
# 查看是否有内容文件
ls content/cn/2025-*/

# 如果没有，从后端拉取
git pull origin main
```

## 🌐 与后端配合

### 后端会做什么

后端（CloudFlare Worker）每天自动：
1. 抓取 Folo AI 资讯
2. 使用 AI 生成日报
3. **推送 Markdown 文件到这个仓库**

推送路径：
- `content/cn/2025-12/2025-12-09.md` （中文）
- `content/en/2025-12/2025-12-09.md` （英文）

### 前端需要做什么

**什么都不用做！** Hugo 会自动：
1. 检测新文件
2. 重新渲染页面
3. 浏览器自动刷新（热重载）

如果想手动拉取最新内容：
```powershell
git pull origin main
```
